FuelFit — AI Provider Setup Guide

How to get an API key and enable AI food photo analysis

100% Transparency: We never want you to be caught by surprise. Here's how FuelFit handles your data:
  • No backend server — all your data stays encrypted on your device only
  • Direct API calls — AI calls go from your device straight to the provider you chose
  • Explicit opt-in — you choose to enable AI during onboarding; you can skip it entirely
  • One toggle for minimizing data exposure — the AI Diet Coach can be turned off in Settings. With it off, your provider is contacted only when you actively take a food photo — your daily nutrition numbers are never sent unprompted for coaching. Photo analysis itself runs only when you choose to take a photo, so it's user-initiated by design.
  • Honest AI label — when coaching falls back to a built-in template (toggle off, no key, or AI failure), the "AI" badge disappears so you always know what's actually generated
  • Export/Import — your data is portable via Settings → Export/Import

Option 1: Qwen / Alibaba Cloud (Default — Cheapest)

Qwen (DashScope) — qwen3-vl-flash

Cheapest · Default

Qwen is FuelFit's default provider. The qwen3-vl-flash model is the cheapest dedicated vision model available and handles food photos especially well, including Chinese cuisine. New Alibaba Cloud accounts get free credits.

Step-by-Step Setup

1
Create an Alibaba Cloud account
Go to account.alibabacloud.com/register (international) or account.aliyun.com (China) and sign up.
2
Activate Model Studio (百炼)
Go to bailian.console.aliyun.com and activate the Model Studio service. New accounts get free credits (1 million tokens) — enough for months of normal use.
3
Create an API key
In the Model Studio console → API Key Management → click "Create API Key" → copy the key (starts with sk-).
4
Enter in FuelFit
Open FuelFit → Settings → AI Services → make sure Qwen is selected (it's the default). Base URL and model arrive pre-filled — bump qwen3-vl-flash to qwen3-vl-plus if you want higher accuracy. Paste your key and tap Verify Connection.

Option 2: OpenAI (Most Reliable)

OpenAI — gpt-4o-mini

Most reliable

OpenAI offers strong reliability and broad ecosystem support. Slightly more expensive than Qwen but extremely predictable behavior — a solid default if you're not in China.

Step-by-Step Setup

1
Create an OpenAI account
Go to platform.openai.com/signup and sign up with your email or Google/Microsoft account.
2
Add a payment method
Navigate to Settings → Billing and add a credit card. You can start with as little as $5.
3
Create an API key
Go to API Keys → click "Create new secret key" → give it a name (e.g., "FuelFit") → click "Create key".
4
Copy and save your key
Copy the key immediately (it starts with sk-). You won't be able to see it again. Store it somewhere safe.
5
Enter in FuelFit
Open FuelFit → Settings → AI Services → select OpenAI. Base URL https://api.openai.com/v1 and model gpt-4o-mini are pre-filled — change them now if you want a different model. Paste your sk-... key, then tap Verify Connection.

Option 4: Custom (Any OpenAI-Compatible Endpoint)

Custom Provider

Power user

If you want to use a provider that isn't built in — Anthropic Claude, Google Gemini, a self-hosted vLLM server, OpenRouter as a multi-provider gateway, or any other OpenAI-compatible API — pick Custom and enter the endpoint yourself.

Why Anthropic isn't built in: Anthropic's official OpenAI-compatible adapter has been unreliable for vision requests (images can be silently dropped). If you want to use Claude for food photos, route through OpenRouter as a Custom provider instead — it's the most reliable path today.

Step-by-Step Setup

1
Get an API key from your chosen provider
For example: OpenRouter, Google AI Studio, or your self-hosted server.
2
Find the OpenAI-compatible Base URL
Most providers document this. Examples: OpenRouter is https://openrouter.ai/api/v1; Google's OpenAI-compatible endpoint is https://generativelanguage.googleapis.com/v1beta/openai.
3
Pick a model that supports vision
FuelFit needs a multimodal model that accepts images. Examples on OpenRouter: anthropic/claude-3.5-sonnet, google/gemini-2.0-flash, openai/gpt-4o-mini.
4
Enter in FuelFit
Open FuelFit → Settings → AI Services → select Custom → fill in Name, Base URL, Model, and API Key → tap Verify Connection.
Cost varies wildly depending on provider and model. OpenRouter publishes a price-per-token table for every model on its platform.

AI Diet Coach Toggle

FuelFit's two AI features have very different data-flow profiles, and only one of them needs a toggle:

  • Photo analysis is user-initiated. Nothing is sent to your provider until you tap the camera button. Don't take a photo, no API call happens. There's nothing to toggle off.
  • The AI Diet Coach runs automatically. When you open the Insights tab or the Home coach card refreshes, FuelFit sends your daily nutrition summary to your provider to generate a coaching message. That's exactly the kind of background call some users don't want — even to a provider they've chosen — for privacy reasons. So this one gets a toggle.

Turning the AI Diet Coach off keeps photo analysis fully working but stops every other unprompted API call. The Insights coach card silently switches to built-in template tips that are computed entirely on your device.

What Each Feature Does

Feature What It Does When It Calls Your Provider How to Stop the Calls
Food Photo Analysis Identifies food items and estimates nutrition from photos you take Only when you tap the camera button Don't take photos. (You can also clear your API key in Settings to break the integration entirely.)
AI Diet Coach Generates coaching messages and daily summaries from your logged data Automatically — when you open Insights, when the Home coach card refreshes, and a few other ambient triggers Toggle off Settings → AI Services → AI Diet Coach. Coach card switches to built-in template tips.

Data Transparency

  • What data is sent where:
  • Food photo analysis: the photo plus a short prompt are sent directly to the AI provider you chose. Nothing else.
  • AI Diet Coach (toggle ON): only your daily nutrition summary numbers are sent — calories, protein, carbs, fat, fiber, sugar, sodium, water, meal count. No photos, no profile details, no historical data beyond today.
  • Template coaching (toggle OFF or AI failure): messages are computed entirely on-device from your logged data. Nothing leaves your phone.

The "AI" Badge — Honest Attribution

On the Insights tab, the coach card shows a small "AI" label only when the message actually came from your AI provider. When the message is built from local templates — because the toggle is off, you have no API key, or the AI call failed — the badge disappears. So you always know whether what you're reading was generated by a model or by FuelFit's own logic.

How to Disable AI Coaching (Settings)

1
Open Settings
Tap the Settings tab in FuelFit.
2
Expand "AI Services"
Tap the AI Services accordion. You'll see your provider configuration and, below it, the AI Diet Coach toggle.
3
Flip the toggle off
With the toggle off you'll see template-based coaching tips instead of AI-generated ones, and the "AI" badge disappears from the Insights coach card. Food photo analysis continues to work normally.

During Onboarding

During the AI setup screen of onboarding, you have two paths:

  • Configure a provider and verify — you'll then see the AI Diet Coach toggle on the next screen, defaulting to ON. You can flip it off there if you only want photo analysis.
  • Skip for now — the AI Diet Coach toggle screen is skipped entirely (with no AI configured, there's nothing to gate). The toggle is set to OFF automatically and a banner reminds you that AI features are disabled. You can configure a provider and turn the toggle back on anytime in Settings → AI Services.
Privacy note: with the AI Diet Coach toggle off, your provider is only contacted when you actively take a food photo. No data is sent for coaching or daily summaries — those are produced by built-in templates that run entirely on your device.

Frequently Asked Questions

Is my API key safe?

Yes. Your AI provider API keys are stored only on your device using the platform’s built-in secure storage — iOS Keychain on Apple devices and Android Keystore-backed encrypted storage on Android (via capacitor-secure-storage-plugin). They are never sent or shared with third parties, or stored in plain text in the app’s local database (IndexedDB or localStorage). Keys are transmitted directly from your device to your chosen AI provider over HTTPS only when you use food photo analysis or AI coach. We never see or collect your data.

What about switching devices?

FuelFit includes a built-in Export/Import function in Settings. Export your data as a file, then import it on your new device. Your meals, settings, and (encrypted) API keys transfer in one step.

Can I use AI for photos only, not coaching?

Yes. Open Settings → AI Services, expand the section, and flip the AI Diet Coach toggle off. This disables AI-generated coaching messages while keeping AI food photo analysis active. The Insights coach card will then show built-in template tips instead, and the "AI" badge will disappear from the card so you can tell at a glance.

Can I use multiple providers?

FuelFit uses one primary provider at a time, but you can configure multiple Custom providers and switch between them in Settings. The three built-in providers (Qwen, OpenAI, Mistral) each remember their own configuration too, so swapping among them is just a tap.

What if I run out of credits?

The app will show an error when you try to analyze a photo or when the AI Diet Coach tries to generate a message. If the coach call fails, the Insights card silently falls back to a template tip (no error toast — and the "AI" badge disappears so you know it's a template). Add credits to your provider account and the next AI call will work again. Your meal history is always saved locally.

Do I need an API key to use FuelFit?

No. FuelFit works without AI for manual meal logging. You only need an API key if you want photo analysis or AI-generated coaching. With no key configured, all coaching uses built-in templates.

Which provider should I choose?

  • Default: Qwen qwen3-vl-flash — cheapest, strong food recognition, free credits for new accounts
  • Most reliable: OpenAI gpt-4o-mini — predictable behavior, broad ecosystem support
  • EU-based: Mistral mistral-small-latest
  • Anything else (Claude, Gemini, self-hosted): use the Custom option, ideally via OpenRouter

How do I switch providers?

Go to Settings → AI Services → tap a different provider in the provider list → enter (or paste) the API key for that provider → tap Verify Connection. Your meal history and AI Diet Coach toggle setting are preserved across the switch.

Why is the model field editable for built-in providers?

Defaults are tuned for cost. If you want higher accuracy you might prefer qwen3-vl-plus on Qwen, gpt-4o on OpenAI, or a different Mistral model. The Base URL and Model fields are pre-filled but editable on every built-in provider — change them upfront and verify, no need to switch to Custom.
