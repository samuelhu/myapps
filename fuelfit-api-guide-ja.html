FuelFit — AIプロバイダー設定ガイド

APIキーを取得し、AI食事写真分析を有効にする方法

100%の透明性: 突然の課金などで驚かせたくはありません。FuelFitはあなたのデータを以下のように扱います：
  • バックエンドサーバーなし — すべてのデータは暗号化され、デバイス内のみに保存されます
  • 直接APIを呼び出し — AIリクエストは、デバイスから選択したプロバイダーへ直接送信されます
  • 明確なオプトイン — オンボーディング時にAIを有効にするか選択でき、スキップすることも可能です
  • データ露出を最小化するスイッチ — 設定でAI食事コーチを無効にできます。無効にすると、写真の撮影時のみプロバイダーにアクセスし、毎日の栄養データがコーチングメッセージの生成目的でバックグラウンド送信されることはありません。写真分析自体は写真を撮影した時のみ実行されるため、ユーザー発信の設計となっています。
  • 誠実なAIラベル — ローカルのテンプレート機能に切り替わった場合（スイッチがオフ、APIキー未設定、AIエラー時）、カードから「AI」バッジが消え、どれが実際に生成されたアドバイスかを常に識別できます
  • エクスポート/インポート — 設定 → エクスポート/インポートから、データを簡単に移行できます

料金はいくらかかりますか？

FuelFitは食事写真の分析にAIを使用します。お客様自身のAPIキーを設定していただく形式（BYOK）をとっており、FuelFit自体へのサブスクリプション料金は一切不要です。費用は選択するプロバイダーと食事の記録頻度によって決定されます。

オプション1: Qwen / Alibaba Cloud (デフォルト — 最安値)

Qwen (DashScope) — qwen3-vl-flash

最安値 · デフォルト

QwenはFuelFitの標準プロバイダーです。qwen3-vl-flashモデルは、画像認識に対応したモデルの中で最も費用が安く、日本の家庭料理やアジアの食事も高精度で識別できます。新規Alibaba Cloudアカウントには無料枠が付与されます。

設定手順

1
Alibaba Cloudアカウントの作成
account.alibabacloud.com/register（インターナショナル版）またはaccount.aliyun.com（中国国内版）で登録を行います。
2
Model Studio (百炼) の有効化
bailian.console.aliyun.com にアクセスしてModel Studioサービスを有効化します。新規アカウントには数ヶ月間の利用に十分な無料枠（100万トークン）が付与されます。
3
APIキーの作成
Model Studioコンソール → API Key Management → "Create API Key" をクリックし、作成されたキー（sk-で始まります）をコピーします。
4
FuelFitアプリに入力
FuelFitを開き、設定 → AIサービス からプロバイダーでQwenを選択します。Base URLとModelは自動入力されます。精度をさらに高めたい場合は、モデルをqwen3-vl-flashからqwen3-vl-plusに変更してください。コピーしたAPIキーを貼り付け、「接続を確認」をタップします。

オプション4: カスタム (すべてのOpenAI互換エンドポイント)

Custom Provider

パワーユーザー向け

Anthropic Claude、Google Gemini、ローカルにホストされたvLLMサーバー、多機能なOpenRouterゲートウェイなど、アプリに組み込まれていないプロバイダーを利用したい場合は、「カスタム」 を選択して手動設定します。

Anthropicが標準搭載されていない理由: Anthropicの公式OpenAI互換アダプターは、画像処理（Vision）リクエスト時に画像をサイレントでドロップしてしまう不具合が確認されています。Claudeを使って食事写真を分析する場合は、OpenRouterをカスタムプロバイダーとして経由するのが最も確実です。

設定手順

1
お好みのプロバイダーからAPIキーを取得
例: OpenRouter、Google AI Studio、またはご自身のローカルサーバーなど。
2
OpenAI互換のBase URLを確認
例: OpenRouterは https://openrouter.ai/api/v1 、Google AI Studioは https://generativelanguage.googleapis.com/v1beta/openai となります。
3
Vision機能対応モデルの選択
画像を受け取れるマルチモーダルモデルが必要です。例: anthropic/claude-3.5-sonnet, google/gemini-2.0-flash, openai/gpt-4o-mini など。
4
FuelFitアプリに入力
FuelFitを開き、設定 → AIサービス から カスタム を選択。名前、Base URL、Model、API Key を手動で入力し、「接続を確認」をタップします。
利用コストはモデルにより大きく異なります。 各モデルのトークン単価は OpenRouter などの公式サイトで公開されています。

AI食事コーチのスイッチについて

FuelFitに搭載された2つのAI機能は、データ送信の仕組みが大きく異なります。そのため、片方にのみオン/オフのスイッチが用意されています：

  • 食事写真分析（ユーザー発信）: カメラで撮影を行い、送信ボタンを押すまでプロバイダーへデータが送られることはありません。写真を撮らなければAPIも呼び出されないため、スイッチ制御は不要です。
  • AI食事コーチ（自動実行）: 「分析」タブを開いた際や、ホーム画面のコーチカードが更新される際に、FuelFitは当日の栄養総計データをプロバイダーに自動送信してアドバイスを生成します。プライバシーの観点から、このようなバックグラウンドでの自動送信を避けたい利用者のために、この機能にはオン/オフのスイッチが設けられています。

AI食事コーチを「オフ」にしても、食事写真のAI分析はそのまま利用可能です。コーチング機能のみがオフになり、アプリ上で動作する内蔵テンプレート（完全ローカル処理）による代替アドバイスに静かに切り替わります。

各機能の特徴とデータ送信タイミング

機能 動作内容 送信タイミング 停止方法
食事写真分析 撮影した写真から食べ物を識別し、栄養成分を推定する カメラボタンをタップして写真を送信したときのみ 写真を撮影しない（または設定からAPIキーを消去して連携を完全に遮断する）
AI食事コーチ 記録された食事データに基づき、コーチングメッセージを生成する 「分析」タブの表示時やホーム画面カード更新時などに自動で送信 スイッチをオフにする 設定 → AIサービス → AI食事コーチ。オフにすると、端末内のローカルテンプレートによるアドバイスに切り替わります。

送信されるデータの内訳

  • 送信先とデータ内容:
  • 食事写真分析: 撮影された写真と「この食べ物を分析して」という指示テキストのみがプロバイダーへ直接送られます。個人情報は含まれません。
  • AI食事コーチ (オンの場合): 当日の栄養素の合計数値（カロリー、タンパク質、炭水化物、脂質、食物繊維、糖分、塩分、水分量、食事回数）のみが送られます。写真やプロフィール、過去の履歴データは一切送信されません。
  • テンプレートコーチ (オフまたは通信エラー時): 端末のローカル環境のみでデータが処理され、一切の情報がスマートフォンから送信されることはありません。

「AI」バッジ — 出所の明記

「分析」タブのコーチカードにおいて、外部プロバイダーのAIによって生成されたアドバイスには、カード内に小さな「AI」ラベルが表示されます。スイッチがオフの場合や、エラーによりローカルテンプレートが作動した場合はこのバッジが非表示になり、ユーザーは読んでいるアドバイスがAI製かローカル製かを一目で確認できます。

AI食事コーチをオフにする方法 (設定)

1
設定タブを開く
FuelFitの下部メニューから「設定」をタップします。
2
「AIサービス」を展開
「AIサービス」のアコーディオンをタップ。プロバイダー設定の下に「AI食事コーチ」のスイッチが表示されます。
3
スイッチをオフにする
オフにすると、アドバイスがローカルテンプレート製に変わり、カード上の「AI」バッジが消えます。写真分析は引き続き正常に利用できます。

オンボーディング中の挙動

初回起動時のAI設定画面では、以下の2つのルートを選べます：

  • プロバイダーを設定し接続確認を行う — 次の画面に「AI食事コーチ」のスイッチ（初期値オン）が表示され、写真分析のみにしたい場合はそこでオフにできます。
  • スキップする — スイッチの選択画面は表示されず、自動的に「オフ」に設定されます。AI機能が無効化されている旨のバナーが表示され、設定からいつでもAPIキーを入力してオンに切り替え可能です。
プライバシーに関する注意: AI食事コーチをオフに設定している場合、あなたがカメラで食事写真を撮影しない限り、プロバイダーと接続されることはありません。日常のまとめデータなどが自動で送信されることはなく、端末内で安全に完結します。

よくある質問 (FAQ)

APIキーは安全に保護されますか？

はい。設定されたAPIキーは端末内に暗号化して保存されます。FuelFitにはバックエンドサーバーが存在しないため、データは他者を経由せず、直接指定のAIプロバイダーへ送信されます。開発元がキーや食事データを閲覧・収集することは技術的に不可能です。

機種変更時のデータ移行はどうなりますか？

設定メニュー内に「インポート/エクスポート」機能を備えています。データをファイルに出力し、新しい端末で読み込ませるだけで、食事データ、設定、さらには暗号化されたAPIキーも一括で移行できます。

コーチ機能は使わず、写真分析のみにAIを使うことはできますか？

はい、可能です。設定 → AIサービスを開き、AI食事コーチのトグルスイッチをオフにしてください。写真分析のVision機能のみがアクティブに維持されます。

複数のプロバイダーを併用できますか？

同時に有効化できるのは1社のみですが、設定画面で簡単に切り替えられます。組み込みの3社（Qwen, OpenAI, Mistral）はそれぞれの設定値を保持するため、ワンタップで切り替え可能です。

プロバイダーのチャージ残高が切れた場合はどうなりますか？

写真の送信時やコーチメッセージの生成時にエラーが表示されます。コーチ呼び出しがエラーになった場合、Insightsカードはエラー表示を出さずにテンプレートでの代用アドバイスを表示し、「AI」バッジが消えます。プロバイダーにチャージすればすぐに復旧します。

FuelFitの利用にAPIキーは必須ですか？

いいえ、必須ではありません。APIキーを設定しなくても、手動での食事記録やカロリー管理は完全無料でご利用いただけます。写真分析やAIコーチを利用したい場合のみキーを設定してください。

どのプロバイダーを選ぶべきですか？

  • デフォルト: Qwen qwen3-vl-flash — 最も低コストで画像認識が優秀、新規アカウントに無料枠あり
  • 信頼性重視: OpenAI gpt-4o-mini — 挙動が最も安定的で指示通りに出力される
  • EU圏ユーザー: Mistral mistral-small-latest
  • その他（ClaudeやGeminiなど）: 「カスタム」を選択し、OpenRouter 経由での接続をお勧めします

プロバイダーの切り替え方は？

設定 → AIサービス に進み、リストから別会社をタップしてAPIキーを入力し、「接続を確認」をタップします。これまでの履歴やスイッチ設定は引き継がれます。

なぜ組み込みプロバイダーのモデル名が編集可能なのですか？

初期設定はコスト効率を最優先したモデルです。より高い認識精度を求める場合は、Qwenの qwen3-vl-plus に書き換えたり、OpenAIの gpt-4o に書き換えて接続確認を行うことができます。
