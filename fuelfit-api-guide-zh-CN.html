如何获取 API 密钥并启用 AI 食物照片分析

100% 透明度：我们不希望您感到意外。以下是 FuelFit 处理您数据的方式：
  • 无后端服务器 — 您的所有数据仅在您的设备上进行加密保存
  • 直接 API 调用 — AI 请求直接从您的设备发送到您选择的服务商
  • 明确选择加入 — 您可以在新手引导期间选择启用 AI，也可以完全跳过它
  • 最小化数据暴露的开关 — 可以在“设置”中关闭 AI 饮食教练。关闭后，只有在您主动拍摄食物照片时才会与服务商联系，您的日常营养数据绝不会被自动发送用于教练分析。照片分析本身仅在您选择拍照时运行，因此完全是由用户启动的。
  • 真实的 AI 标签 — 当教练建议回退到内置模板（开关关闭、无密钥或 AI 失败）时，“AI”标记会消失，以便您始终清楚哪些内容是真正生成的
  • 导入/导出 — 您的数据可以通过“设置 → 导入/导出”进行便携迁移

这需要花多少钱？

FuelFit 使用 AI 分析您的食物照片。您需要自备 API 密钥 (BYOK) — FuelFit 本身不收取任何订阅费用。费用完全取决于您选择的服务商以及您记录饮食的频率。

AI 饮食教练开关

FuelFit 的两个 AI 功能具有非常不同的数据流配置，且只有其中一个需要开关：

  • 食物照片分析是由用户发起的。 在您主动点击相机按钮拍照之前，没有任何数据会被发给服务商。如果您不拍照，就不会发起任何 API 调用，因此这里无需开关。
  • AI 饮食教练是后台自动运行的。 当您打开“分析”选项卡或“今天”选项卡上的教练卡片刷新时，FuelFit 会自动将您的每日营养数据发送给服务商以生成针对性的建议。出于隐私原因，有些用户可能不希望进行后台自动调用（即使是发给他们自己选择的服务商）。因此，这个功能配有专门的开关。

关闭 AI 饮食教练将保持食物照片分析功能完全正常，但会停止所有其他后台自动 API 调用。Insights 中的教练卡片将静默切换为完全在您设备上计算的内置模板建议。

各个功能的数据流差异

功能名称 功能说明 何时调用您的 AI 服务商 如何停止调用
食物照片分析 识别食物成分并从拍照中估算营养数值 仅在您点击相机按钮拍照时 不拍照即可。（您也可以在设置中清除 API 密钥以完全断开集成。）
AI 饮食教练 根据您的餐食记录数据生成教练反馈和每日总结 后台自动调用 — 打开分析选项卡、首页卡片刷新及其他环境触发时 关闭开关 设置 → AI 服务 → AI 饮食教练。教练卡片将切换为内置本地模板建议。

数据透明度

  • 具体数据流向：
  • 食物照片分析： 仅将照片和简短提示词直接发送至您选择的 AI 服务商。没有其他个人数据。
  • AI 饮食教练（开关开启）： 仅发送您每日的营养总和数值 — 卡路里、蛋白质、碳水化合物、脂肪、膳食纤维、糖、钠、水分和餐食数量。不发送照片，不发送个人资料，不发送今天以外的历史数据。
  • 本地模板教练（开关关闭或 AI 调用失败）： 建议完全由本地设备根据您的记录数据计算得出。没有任何数据离开您的手机。

“AI” 标签 — 诚实归属

在“分析”选项卡上，教练卡片仅在消息确实来自您的 AI 服务商时才显示“AI”标签。如果消息是根据本地模板生成的（因为开关关闭、没有配置 API 密钥或 AI 调用失败），该标签会消失。因此，您始终能够分辨所读内容是由模型生成的还是由 FuelFit 自身逻辑生成的。

如何在设置中禁用 AI 教练

1
打开设置
点击 FuelFit 底部的“设置”选项卡。
2
展开“AI 服务”
点击“AI 服务”折叠面板。您将看到服务商配置以及下方的“AI 饮食教练”开关。
3
关闭该开关
关闭开关后，您将看到基于本地模板的教练建议，且“AI”标签会从卡片中消失。食物照片分析仍可正常工作。

在新手引导期间

在新手引导的 AI 设置屏幕中，您有两个选择：

  • 配置并验证服务商 — 之后您将在下一个屏幕上看到“AI 饮食教练”开关，默认开启。如果您只想要照片分析，可以在此关闭它。
  • 稍后设置 — 将完全跳过“AI 饮食教练”开关屏幕（没有配置 AI 就不需要控制）。开关将自动设为关闭，并且会有横幅提示您已禁用 AI 功能。您随时可以在设置中配置服务商并重新开启。
隐私说明：当 AI 饮食教练开关关闭时，只有在您主动拍摄食物照片时才会与服务商建立连接。教练或每日总结绝不会向服务商发送任何数据 — 这些都是由完全运行在您设备上的内置模板生成的。

常见问题解答

我的 API 密钥安全吗？

安全。您的 AI 服务商 API 密钥仅存储在您的设备上，使用平台内置的安全存储——Apple 设备上的 iOS Keychain 和 Android 设备上的受 Android Keystore 支持的加密存储（通过 capacitor-secure-storage-plugin 插件）。它们绝不会发送或共享给第三方，也不会以明文形式存储在应用程序的本地数据库（IndexedDB 或 localStorage）中。只有在您使用食物照片分析或 AI 饮食教练时，密钥才会通过 HTTPS 直接从您的设备传输给您选择的 AI 服务商。我们永远不会收集或查看您的数据。

更换设备怎么办？

FuelFit 在“设置”中提供了内置的“导入/导出”功能。在一台设备上导出数据为文件，然后在另一台设备上导入即可。您的餐食记录、设置以及（加密的）API 密钥会一步完成迁移。

我可以只把 AI 用于照片，而不用于教练吗？

可以。打开设置 → AI 服务，展开该部分，然后关闭 AI 饮食教练开关。这将禁用 AI 生成的教练建议，同时保持 AI 食物照片分析处于活动状态。分析卡片随后将显示内置模板建议，且“AI”标签将消失。

我可以使用多个服务商吗？

FuelFit 一次使用一个主要服务商，但您可以在“自定义”选项中配置其他兼容的 API 并在设置中切换。三个内置服务商（Qwen、OpenAI、Mistral）也会分别记住它们自己的配置，因此只需点按一下即可切换。

如果余额不足会怎样？

当您尝试分析照片或 AI 饮食教练尝试生成建议时，应用会显示错误。如果教练调用失败，分析卡片会静默退回到模板建议（不会弹出错误提示，且“AI”标签消失以防误导）。给您的服务商账户充值后，下一次 AI 调用就会重新工作。您的本地餐食记录始终被安全保存。

使用 FuelFit 必须要有 API 密钥吗？

不需要。FuelFit 无需 AI 即可进行手动饮食记录。只有在您需要照片分析或 AI 生成的教练建议时才需要 API 密钥。如果没有配置密钥，所有教练都将使用本地内置模板。

我应该选择哪个服务商？

  • 默认：Qwen qwen3-vl-flash — 最实惠，食物识别强，新账户有免费额度
  • 最可靠：OpenAI gpt-4o-mini — 表现可预测，生态支持广泛
  • 总部位于欧盟：Mistral mistral-small-latest
  • 其他任何（Claude、Gemini、自建）：使用“自定义”选项，推荐通过 OpenRouter 路由

如何切换服务商？

前往设置 → AI 服务 → 在列表中选择其他服务商 → 输入（或粘贴）该服务商的 API 密钥 → 点击“验证连接”。您的餐食记录和 AI 饮食教练开关设置将完整保留。

为什么内置服务商的模型字段是可以编辑的？

默认值是基于性价比调优的。如果您想要更高的准确度，可以在验证前将 Qwen 的 qwen3-vl-flash 改为 qwen3-vl-plus，或者在 OpenAI 上改为 gpt-4o。内置服务商的 Base URL 和 Model 字段都是可编辑的，无需切换到“自定义”即可修改。
